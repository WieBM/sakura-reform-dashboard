## 🔴 High — 5件

### BUG-01: ダッシュボード再訪時にチャートがクラッシュ — "Canvas is already in use"

**ページ**: ダッシュボード  
**カテゴリ**: Frontend JS

**原因**  
案件/請求 CRUD 後に `loaded['dashboard']` が `false` にリセットされ、再訪時に同じ `<canvas>` に対して `new Chart(...)` が再実行される。既存インスタンスを `.destroy()` せずに再生成するため Chart.js が例外を投げる。

**再現手順**
1. ダッシュボードを開き、チャートが表示されることを確認する
2. 案件管理で任意の案件を新規追加または編集・保存する
3. サイドバーからダッシュボードに戻る
4. コンソールに `"Canvas is already in use. Chart with ID '0' must be destroyed…"` が出力される

**修正案**  
チャートインスタンスをモジュールスコープ変数に保持し、`new Chart()` 前に `.destroy()` を呼ぶ。

```js
let chartStatusInst = null;

// loadDashboard() 内:
if (chartStatusInst) chartStatusInst.destroy();
chartStatusInst = new Chart(
  document.getElementById('chartStatus'), { ... }
);
```

`chartTypeInst`・`chartMonthlyInst` も同様。

### BUG-02: モバイルのハンバーガーボタンにイベントリスナーが未設定 — メニュー操作不能

**ページ**: 全ページ (モバイル)  
**カテゴリ**: Mobile UX

**原因**  
`#hamburger` ボタンに `click` イベントリスナーが一切なく、タップしても何も起きない。加えて `.topbar` の CSS がデフォルト `display: none` のまま、どのメディアクエリでも `display: flex` に上書きされないため、ボタン自体が表示されない（BUG-10 と連動）。

**修正案**

```js
// JS — サイドバー初期化付近に追加
document.getElementById('hamburger')
  .addEventListener('click', openSidebar);
```

```css
/* CSS — @media (max-width: 480px) ブロックに追加 */
.topbar { display: flex; }
.sidebar { display: none; }
.main { margin-left: 0; padding-top: 68px; }
```

### BUG-03: `loadEmployees()` の async レースコンディション — `loaded` フラグがフェッチ完了前にセットされる

**ページ**: 社員管理  
**カテゴリ**: Frontend JS

**原因**  
社員 CRUD 保存後のコードが `loadEmployees()` を呼んだ直後（非同期完了前）に `loaded['employees'] = true` を同期的にセットする。低速ネットワークや高負荷時、フェッチ完了前にユーザーが画面を切り替えると、次の訪問で `loaded` が `true` のまま読み込みがスキップされ古いデータが表示される。

**修正案**  
手動の `loaded['employees'] = true` 行を削除し、`loadEmployees()` 内部のフェッチ完了時に設定するか、呼び出し側で `await` する。

```js
// CRUD 保存後:
allEmployees = [];
await loadEmployees(); // loaded フラグは関数内部で管理
```

### BUG-04: 数値フィールドが 0 の場合、編集モーダルで値が消える — データ上書き危険

**ページ**: 案件管理 / 請求・入金 / 顧客台帳  
**カテゴリ**: Data / UX

**原因**  
`p.estimate_amount || ''` のように falsy 判定しているため、値が `0` の場合 `0 || '' → ''` となり空欄になる。そのまま保存すると `null` がサーバーに送信される。

**影響フィールド**  
`estimate_amount`・`contract_amount`（案件）、`amount`（請求）、`age_years`（顧客）

**修正案**

```js
// 変更前:
input.value = p.estimate_amount || '';

// 変更後:
input.value = (p.estimate_amount !== null && p.estimate_amount !== undefined)
  ? p.estimate_amount : '';
```

### BUG-05: 社員編集モーダルを開くたびに `/api/employees` を再フェッチ — キャッシュ未使用

**ページ**: 社員管理  
**カテゴリ**: Performance

**原因**  
`openEmployeeModal(id)` が毎回 `fetch('/api/employees')` を呼ぶ。顧客・案件・請求の各モーダルはメモリキャッシュ（`allCustomers` 等）から検索するが、社員モーダルのみ例外的に毎回全件取得している。

**修正案**  
`loadEmployees()` 実行時に `allEmployeesFull = emps` として保持し、モーダル内では `allEmployeesFull.find(x => x.id === id)` で参照する。

## 🟠 Medium — 5件

### BUG-06: ユーザー入力値が HTML エスケープなしで `insertAdjacentHTML` に挿入 — Stored XSS

**ページ**: 案件管理 / 顧客台帳 / 請求・入金 / 社員管理  
**カテゴリ**: Security

**原因**  
`p.name`・`c.name`・`e.name` 等がテンプレートリテラルでそのまま HTML に挿入される。`<img src=x onerror=alert(1)>` を案件名に保存するとスクリプトが実行される。既存の `esc()` はシングルクォートのみエスケープするため不十分。

**修正案**

```js
function html(str) {
  return (str || '')
    .replace(/&/g, '&amp;')
    .replace(/</g,  '&lt;')
    .replace(/>/g,  '&gt;')
    .replace(/"/g,  '&quot;');
}
// 使用例: <td>${html(p.name)}</td>
```

全テーブルセルの `insertAdjacentHTML` で `html()` を適用。

### BUG-08: `prj-status` がフィールドリセット対象に含まれない — 前回の値が残る

**ページ**: 案件管理  
**カテゴリ**: Bug

**原因**

### BUG-09: 全 `fetch()` 呼び出しにエラーハンドリングなし — ネットワーク障害時にサイレント失敗

**ページ**: 全ページ  
**カテゴリ**: Robustness

**原因**  
全ての `fetch()` に `.catch()` がなく、サーバー停止やネットワーク断時に Unhandled Promise Rejection が発生。テーブルは空のままエラーメッセージも表示されない。`loadDashboard()` の `Promise.all()` も同様。

**修正案**

```js
async function loadProjects() {
  try {
    allProjects = await fetch('/api/projects').then(r => r.json());
    renderProjects();
  } catch (err) {
    showToast('案件データの取得に失敗しました', 'error');
  }
}
```

全ページロード関数と CRUD submit ハンドラに同様の `try/catch` を追加。

### BUG-10: `.topbar` が全ブレークポイントで `display: none` のまま — モバイルナビ完全非表示

**ページ**: 全ページ (モバイル)  
**カテゴリ**: Mobile UX

**原因**  
デフォルト CSS に `.topbar { display: none }`。`@media (max-width: 1100px)` ブロックも同じく `display: none` を再定義。`display: flex` に切り替えるメディアクエリが存在しないため、いかなる画面サイズでもトップバーは表示されない。BUG-02 と合わせて修正が必要。

**修正案**

```css
@media (max-width: 480px) {
  .topbar  { display: flex; }
  .sidebar { display: none; }
  .main    { margin-left: 0; padding-top: 68px; }
}
```
